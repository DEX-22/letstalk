# Development Rules

- Every pull request must include tests.
- All code must pass TypeScript strict checking.
- Components must be tested with Vitest.
- E2E flows must be tested with Playwright.
- No console.log in production code.
- All text must be wrapped in i18n-compatible format (future).
- Follow accessibility AA standards.
- Mobile-first responsive design.
- Dark mode is the default theme.
- No dead code or commented-out code.
